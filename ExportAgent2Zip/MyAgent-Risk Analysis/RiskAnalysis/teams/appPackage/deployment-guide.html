Quick Start

You need two values from your environment:

ValueWhere to Find ItExample
Entra App ID Azure Portal → Entra ID → App registrations → your PostureIQ app → Application (client) ID ffb6f10d-6991-430e-b3d6-23a0101a92b1
Backend FQDN Azure Portal → Container Apps → your app → Application Url (without https://) myapp.delightfulbush-cc9cf399.northeurope.azurecontainerapps.io

Then run:

# Build the package (resolves placeholders, validates, creates PostureIQ.zip)
.\Deploy-TeamsApp.ps1 -EntraAppId "<YOUR_APP_ID>" -BackendFqdn "<YOUR_FQDN>" -Mode Build

# Deploy — pick one:
.\Deploy-TeamsApp.ps1 -EntraAppId "<YOUR_APP_ID>" -BackendFqdn "<YOUR_FQDN>" -Mode Sideload     # personal
.\Deploy-TeamsApp.ps1 -EntraAppId "<YOUR_APP_ID>" -BackendFqdn "<YOUR_FQDN>" -Mode OrgPublish   # org-wide
Template Manifest The manifest.json in this folder uses placeholders {{TEAMS_APP_ID}} and {{BACKEND_FQDN}}. The deployment script resolves them automatically into a build/ subfolder — the template is never modified.

Prerequisites

  1. Entra App Registration A registered app in Entra ID with correct redirect URIs for your PostureIQ backend. Note the Application (client) ID — this is your {{TEAMS_APP_ID}}.
  2. Backend Running Your PostureIQ Container App must be deployed and healthy. Test: 
    curl https://<YOUR_FQDN>/health   # expect 200 OK
  3. Custom App Uploads Enabled (Path A only) Teams Admin Center → Teams apps → Permission policies → "Upload custom apps" = On.
    Path B works even if sideloading is disabled.
  4. PowerShell 5.1+ (for script) Optionally install Microsoft.Graph.Teams for automated deployment: 
    Install-Module Microsoft.Graph.Teams -Scope CurrentUser
    Without it, the script builds the zip and prints manual upload steps.

Deployment Script

The included Deploy-TeamsApp.ps1 handles everything: resolves placeholders, validates, builds, and deploys.

Step 1 — Validate (dry run, no changes)
.\Deploy-TeamsApp.ps1 `
    -EntraAppId "<YOUR_APP_ID>" `
    -BackendFqdn "<YOUR_FQDN>" `
    -Mode Validate
Step 2 — Build PostureIQ.zip
.\Deploy-TeamsApp.ps1 `
    -EntraAppId "<YOUR_APP_ID>" `
    -BackendFqdn "<YOUR_FQDN>" `
    -Mode Build
Step 3 — Deploy (choose one)
# Path A — personal sideload
.\Deploy-TeamsApp.ps1 -EntraAppId "<YOUR_APP_ID>" -BackendFqdn "<YOUR_FQDN>" -Mode Sideload

# Path B — submit to org
.\Deploy-TeamsApp.ps1 -EntraAppId "<YOUR_APP_ID>" -BackendFqdn "<YOUR_FQDN>" -Mode OrgPublish

What the Script Does

PhaseAction
0. ResolveReads template manifest.json, replaces {{TEAMS_APP_ID}} and {{BACKEND_FQDN}}, writes resolved files to build/ — template stays untouched
1. ValidateChecks JSON validity, no leftover placeholders, App ID match, icon files, backend health
2. BuildCreates build/PostureIQ.zip from resolved manifest + icons, verifies zip contents
3a. SideloadGraph API install for current user, or manual steps
3b. OrgPublishGraph API submit with requiresReview=true, or manual steps
Custom Output Directory Use -OutputDir "C:\my\path" to place the built zip elsewhere. Defaults to teams/appPackage/build/.

Common Manual Steps (Both Paths)

If you prefer to upload manually (or if the Graph module is not installed), steps 1–4 are the same for both paths:

  1. Open Microsoft Teams — Desktop app or teams.microsoft.com.
  2. Navigate to Apps — Click Apps in the left sidebar.
  3. Manage Your Apps — Click "Manage your apps" (bottom-left).
  4. Click "Upload an app" — In the toolbar. You'll see two options — choose based on your path.

Path A — Upload a Custom App

Personal / Testing Install for your account only. No admin approval needed.

Step-by-Step Walkthrough

  1. Open Microsoft Teams Launch the Teams desktop app or go to teams.microsoft.com.
  2. Navigate to Apps Click Apps in the left sidebar (the grid/store icon near the bottom).
  3. Open "Manage your apps" At the bottom-left of the Apps page, click "Manage your apps". This opens the "Installed apps" management view.
  4. Click "Upload an app" In the toolbar (top area), click the "Upload an app" button. A dropdown appears with two options.
  5. Select "Upload a custom app" Click the first option — "Upload a custom app". A file browser dialog opens.
  6. Select PostureIQ.zip Browse to build/PostureIQ.zip (generated by the deployment script) and click Open.

Path A — Approval & Consent Flow

After selecting PostureIQ.zip, Teams processes the package and shows an App Review Card. Here's what to expect:

Verify Before Proceeding If any fields look wrong (e.g., unresolved {{BACKEND_FQDN}} in descriptions, or wrong app name), click Cancel and re-run -Mode Validate to diagnose.

Click "Add"

  1. Review the card Verify the app name is PostureIQ. Check the icon and description match expectations.
  2. Click "Add" At the bottom of the review card, click the blue "Add" button. This installs the app for your account only — no org-wide impact.

Post-Install: First Launch & Authentication

  1. PostureIQ opens automatically Teams navigates to PostureIQ. You'll see the 5 personal tabs in the header:
    Portal
    Security Assessment
    Risk Analysis
    Data Security
    Cloud Explorer
  2. MSAL sign-in popup On first launch, MSAL.js triggers a sign-in popup window. Sign in with your organizational account (the same tenant as the Entra app registration).
    Consent Prompt If this is the first user to use the app, you may see a permissions consent dialog listing the scopes your PostureIQ backend requires (e.g., User.Read, Microsoft Graph API scopes). Click "Accept" to grant consent. If your admin requires admin consent, contact your Teams/Global Administrator.
  3. Verify functionality After sign-in, the Portal tab should load your PostureIQ dashboard. Verify:
    • Dashboard loads without blank/white screen
    • AI chat widget is responsive
    • Each tab navigates to the correct backend page
    • No CORS errors in the browser console (F12)

Path B — Submit an App to Your Org

Organization-Wide Submit for admin approval, then available to all users.

Part 1 — Submit (Any User)

  1. Select "Submit an app to your org" — The second option in the dialog.
  2. Select the zip file — Browse to build/PostureIQ.zip.
  3. Confirm — App enters Pending approval. Check status under "Pending requests".

Part 2 — Admin Approval (Teams Admin)

  1. Open Teams Admin Center — admin.teams.microsoft.com
  2. Manage apps — Teams apps → Manage apps. Find PostureIQ (status: Submitted).
  3. Publish — Click PostureIQ, review, click "Publish".
  4. Pin (optional) — Setup policies → Global policy → Pinned apps → add PostureIQ.
Admin Role Required Steps 4–7 require Teams Administrator or Global Administrator.

Blank white pageBothBackend down or CORS missing teams.microsoft.com.
Sign-in popup blockedBothAllow popups for login.microsoftonline.com and your FQDN.
